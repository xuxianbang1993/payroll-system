**Tech Stack:** TypeScript strict + decimal.js（金额精度） + Vitest（单元测试） + React 19 + shadcn/ui + Tailwind CSS + Zustand + i18next + Playwright（E2E）

### 2.2 类型定义（`src/types/voucher.ts`）

```typescript
/** 凭证分录方向 */
export type EntryDirection = "debit" | "credit";

/** 单条凭证分录 */
export interface VoucherEntry {
  direction: EntryDirection;
  account: string;        // 会计科目（中文全称）
  amount: number;         // 金额（已 round(2)）
}

/** 单张凭证 */
export interface Voucher {
  id: number;             // 凭证序号（1-5）
  title: string;          // 凭证标题（含 YYYY-MM）
  entries: VoucherEntry[];// 分录列表（已过滤掉 amount=0 的分录）
  debitTotal: number;     // 借方合计
  creditTotal: number;    // 贷方合计
  balanced: boolean;      // |debitTotal - creditTotal| < 0.01
}

/** 完整凭证集（5 张） */
export interface VoucherSet {
  month: string;          // YYYY-MM
  vouchers: Voucher[];    // 仅包含有效凭证（全部分录为 0 的凭证不包含）
  balanceCheck: {
    salaryPayableCredit: number;   // 应付职工薪酬-人员工资 贷方
    salaryPayableDebit: number;    // 应付职工薪酬-人员工资 借方
    salaryPayableBalance: number;  // 余额（应为 0）
    balanced: boolean;             // |余额| < 0.01
  };
}
```

### 2.3 生成器设计（`src/services/voucherGenerator.ts`）

**核心签名：**

```typescript
export function generateVouchers(agg: AggregateResult, month: string): VoucherSet
```

**内部结构（积木式 — 每张凭证一个独立纯函数）：**

```
generateVoucher1(agg, month) → Voucher | null   // 计提月工资
generateVoucher2(agg, month) → Voucher | null   // 缴纳社保
generateVoucher3(agg, month) → Voucher | null   // 支付公积金
generateVoucher4(agg, month) → Voucher | null   // 缴纳个税
generateVoucher5(agg, month) → Voucher | null   // 发放工资
```

**规则：**
- 全部金额使用 `decimal.js`（与 calculator/aggregator 一致）
- 每张凭证独立生成，返回 `null` 表示全部分录为 0
- `generateVouchers` 过滤 null 后组装 VoucherSet
- 余额校验：计算 "应付职工薪酬-人员工资" 的借贷净额

## 10. 风险与缓解

| 风险 | 概率 | 缓解 |
|------|------|------|
| 凭证 ① 分录逻辑复杂，容易遗漏 | 中 | 严格对照 PRD 06 §2 三部分，逐条实现+测试 |

## 11. 架构决策（已锁定）

| 决策 | 选择 | 理由 |
|------|------|------|
| 凭证数据结构 | `VoucherEntry[] → Voucher → VoucherSet` 三层 | 最小化耦合，组件只需 `Voucher`，页面需 `VoucherSet` |
| 生成函数返回 `null` | 全部分录为 0 时返回 null | 比返回空数组语义更明确，页面直接 filter(Boolean) |
| 科目名称硬编码中文 | 会计科目是领域术语，不走 i18n | 会计准则中科目名称是固定的 |
| 不新建 voucher-store | 凭证是从 payroll slips 实时计算的，无独立状态 | 避免状态冗余，遵循 YAGNI |
| CSV 不用 SheetJS | 凭证 CSV 结构简单（5 列），无需引入重量级库 | 减少依赖，YAGNI |
| 公司筛选在页面层 | `aggregatePaySlips(slips, filterCompany)` 已支持 | 复用 P2.2 能力，不重复实现 |
